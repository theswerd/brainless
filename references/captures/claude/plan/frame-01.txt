   Plan: Dark-Mode Toggle on Settings Page

   Add a dark-mode toggle to the settings page by introducing a theme state with three values
   (light, dark, system) managed in a small theme provider at the app root, which applies a dark
   class (or data-theme attribute) to the document root element so styles can key off it. Persist
   the user's choice in localStorage and, on first load, fall back to the prefers-color-scheme media
   query when the stored value is "system" or absent; apply the theme in an inline script before
   first paint to avoid a flash of the wrong theme. On the settings page, render a labeled,
   keyboard-accessible toggle (or three-way segmented control) wired to the provider, and verify by
   switching themes, reloading the page, and changing the OS-level preference to confirm persistence
   and system-following behavior both work.
